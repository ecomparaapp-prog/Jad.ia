Prompt: Implementação de GitHub OAuth para Usuários Externos
"Implemente o fluxo de Autenticação GitHub OAuth na Jad.ia para que usuários finais possam conectar suas próprias contas e salvar projetos diretamente.

1. Configuração do Fluxo OAuth (Backend):

Crie uma rota de autenticação /auth/github e uma rota de callback /auth/github/callback.

Utilize a biblioteca axios ou octokit para trocar o 'code' temporário do GitHub por um access_token do usuário.

Importante: Não armazene o token no localStorage de forma insegura. Utilize cookies de sessão ou um banco de dados temporário vinculado à sessão do usuário.

2. Variáveis de Ambiente do App (Developer Settings):

O sistema deve utilizar process.env.GITHUB_CLIENT_ID e process.env.GITHUB_CLIENT_SECRET (que eu configurarei nos Secrets do Replit) para identificar a Jad.ia perante o GitHub.

3. Interface de Conexão (Frontend):

Adicione um botão estilizado no cabeçalho ou na barra lateral: 'Conectar meu GitHub'.

Após a conexão, substitua o botão pelo avatar do usuário e o status 'Conectado'.

Adicione um botão [ Publicar no meu GitHub] que só fica ativo após a conexão.

4. Lógica de Criação de Repositorio por Usuário:

Quando o usuário clicar em publicar, a Jad.ia deve usar o token dele para:

Criar um novo repositório com o nome do projeto atual.

Fazer o upload de todos os arquivos gerados (HTML, CSS, JS, App.js, etc.).

Gerar um commit inicial: 'Project generated by Jad.ia '.

5. Segurança:

Garanta que o escopo (scope) solicitado ao usuário seja apenas public_repo ou repo para que a Jad.ia tenha permissão apenas de gerenciar os códigos, sem acessar dados privados desnecessários."

🛠️ O que você precisa fazer no GitHub (Passo a Passo):
Como você é o "Dono" da Jad.ia, você precisa registrar a aplicação no GitHub para receber o ID e o Secret:

Vá em GitHub Settings > Developer Settings > OAuth Apps > New OAuth App.

Application Name: Jad.ia.

Homepage URL: A URL do seu projeto no Replit (ex: https://jadia.seu-usuario.repl.co).

Authorization callback URL: A mesma URL acima + /auth/github/callback.

Clique em Register application.

Copie o Client ID e gere um Client Secret.

Coloque ambos nos Secrets do seu Replit com os nomes: GITHUB_CLIENT_ID e GITHUB_CLIENT_SECRET.

Por que isso é melhor?
Profissionalismo: O usuário não precisa saber o que é um "Token" ou "API Key". Ele apenas clica em "Autorizar".

Escalabilidade: Milhares de usuários podem usar a Jad.ia simultaneamente, cada um salvando no seu próprio perfil.

Segurança: Você nunca terá acesso à senha do usuário, apenas uma permissão temporária para criar o repositório do projeto.
